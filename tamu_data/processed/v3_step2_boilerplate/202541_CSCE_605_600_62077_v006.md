[16.0pt] Course Description
Compiler Design. (3-0). Credit 3. Advanced topics in compiler writing; parser
generators and compiler-compilers; dynamic storage and scope resolution; data
flow analysis and code optimization. Prerequisite: CSCE 434.
[16.0pt] Course Prerequisites
Prerequisite/Corequisite(s):  CSCE 434.

knowledge is expected. Sufficient programming experience and background are

environments, executing the compilers, and running compiled programs (with or

[16.0pt] Course Learning Outcomes
Upon completion of this course, the student will be able to:
Understand the translation process from user code to machine code.
Describe major components of a typical modern compiler and their function.
Apply parsing techniques such as LL(1), SLR(1), and LR(1).
Design and implement sound dataflow analyses.
Understand and implement code optimization techniques.
Develop a front end and a back end of an optimizing compiler that can
generate executable machine code from source code.

[12.9pt] Project Logistics
The language used for the project is Java. No other language is allowed.
For the first 2 project assignments, students must work individually.
From the third project assignment till the last (6th), students must work in
pairs, from the same section.
Students may choose to work individually, but should be aware that the
project’s requirement is not reduced proportionally.
Students may break from the pair only after the third project assignment, but
must find themselves a new partner for the last three project assignments.
After the fourth assignment, if the pair structure still does not work for a
student, that student goes solo.
Public testcases requirement:
Each student must contribute at least 1 test to the set of public testcases for each
project assignment to a shared GitHub repository. Each test is required to satisfy 3
requirements:
1. Unique: the test must be reasonably different from existing tests.
2. On time: the test must be committed and pushed to the shared repository 72
hours before the assignment deadline. The cut-off for test submission is 24
hours before the assignment deadline.
3. Usable: the test must have no inconsistency. It is the author’s responsibility to
fix issues by the cut-off.
5% of full credit penalty each if the test is duplicated, late, or inconsistent; the
penalty can be applied at any time after the deadline.
No test submission penalty: 17% of full credit
Project assignments grading:
Each front-end project assignment (PA 1-3) grade:
20%: passing 80% of public tests
80%: passing 100% of private tests
Back-end project assignments (PA 4-6) will be graded as a whole at the end of
the semester. Further details of the grading are in the project description.

[12.9pt] Plagiarism
Plagiarism is one of the worst academic sins, for the plagiarist destroys trust among
colleagues and the purpose of learning. Plagiarism is commonly defined as passing
off as one’s own ideas, words, writings, code, etc., which belong to another person.
By this definition, in this course, a student commits plagiarism if copying the work
elsewhere, even with the permission of the author, and turns it in as one’s own.
The instructor encourages collaboration and the exchange of ideas among
students. Collaboration is important for facilitating learning, and peers often can be
a great resource. However, all assignments in this course are individual efforts. All
assignments must be fully completed by only the student to whom they are
assigned, without unauthorized aid. Any academic dishonesty will not be tolerated
and will be severely dealt with. Consequences include a zero grade for the
activity/assignment, course failure, and dismissal from the university, among others.
Refer to the Academic Integrity Statement and Policy below. All involved parties will
be penalized equally.
Specifically, students may discuss the assignments at a high level (e.g.,
concepts, approaches) with friends but absolutely no actual solution sharing
(voluntarily or involuntarily) is allowed.
Code: Students are not allowed to use code found elsewhere (e.g., Google
searches, online repositories, past offerings of the course). The use of a
private code repository, with a detailed commit history, is recommended and
may be requested at any time to answer authorship inquiries. Students also
must not attempt to circumvent the objective of an assignment (e.g., using a
banned library or a function to bypass steps outlined in the assignment)
Non-code: The write-up of homeworks must be done individually and must
not be shared in any way or form.
No full or partial deliverables are allowed to be knowingly or unknowingly
released in the public domain even after the deadlines and after the course is
over. Students are solely responsible for securing all deliverables away from
the public.
Group review sessions for exams are allowed and encouraged.
Posting assignments to or copying solutions from unauthorized sources is
academic misconduct.
There have been past incidents where students used code from Chegg or
similar tutoring websites; the use of such resources is not allowed.
Complicity
Every student should understand that complicity – helping or attempting to
help another student commit an act of academic dishonesty – also
constitutes academic dishonesty and carries the same punishment as
cheating and plagiarism.
In other words, if providing a solution to another student, even if that student
does not turn it in for credit, the provider has committed an act of academic
dishonesty.
Occasionally, we find evidence of plagiarism after the assignments have been
graded. For example, a prior assignment may be flagged when a new detector
becomes available. For these reasons, you should treat your assignment grades as
provisional. Any prior submission that is later found to constitute a violation will be
reported to the Aggie Honor System Office (AHSO).
