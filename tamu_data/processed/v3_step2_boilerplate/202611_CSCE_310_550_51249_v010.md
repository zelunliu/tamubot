[16.0pt] Catalog Description
Database Systems. (3-0). Credit 3. File structures and access methods; database
modeling, design and user interface; components of database management
systems; information storage and retrieval, query languages, high-level language
interface with database systems. Prerequisites: CSCE 221 with a grade of C or
better; junior or senior classification.
[16.0pt] Course Prerequisites
Prerequisite/Corequisite(s):  CSCE 221 with a grade of C or better; junior or senior
classification.
[16.0pt] Course Learning Outcomes
Upon completion of this course, the learner will be able to:
Explain the fundamental concepts of database systems, including data
models, database architecture, and the role of database management
systems (DBMS)
Design and implement Entity-Relationship (ER) and Enhanced Entity-
Relationship (EER) models to represent real-world data requirements
Apply normalization theory and functional dependencies to design efficient
and well-structured relational database schemas
Write complex SQL queries to create, manipulate, and retrieve data from
relational databases
Evaluate and compare different database paradigms, including relational
databases, NoSQL systems, and distributed databases
Analyze data warehouse architectures and understand their application in
business intelligence and decision support systems
Implement database connectivity in applications and apply security principles
to protect database systems

[16.0pt] AI Statement
With the rise of artificial intelligence (AI) technologies, programming and problem-
solving practices are evolving. While these tools can assist with brainstorming,
debugging, or exploring alternative approaches, they do not replace the essential
skills of designing, writing, and testing your own code. Nor do they substitute for the
critical thinking required to analyze problems, evaluate solutions, and apply course
concepts independently.
In this course, you are permitted to use generative AI tools (e.g., ChatGPT, GitHub
Copilot, Google Gemini) to support your learning, under the following conditions:
AI may be used as a supplement, not a substitute.  You may consult AI for
hints, debugging suggestions, or explanations, but you are expected to write
and understand your own code.
You must cite AI assistance.  If AI tools inform your work (e.g., generating a
code snippet, suggesting an algorithm, or helping you debug), you must
clearly document this in comments within your code (e.g.,  # AI tool
(ChatGPT) suggested this loop structure   ).
AI should not complete your assignments for you.  Submitting work
primarily written by AI is considered academic dishonesty. You are
responsible for understanding and being able to explain every line of code
you submit.
Limit reliance.  No more than 30% of an assignment's code may be directly
AI-generated. The majority of the work must reflect your own effort and
understanding.
Important reminders:
Generative AI may produce incorrect or inefficient code. Always test and
verify any AI-assisted solution before using it.
AI tools may generate code that does not follow best practices or course
requirements. You are responsible for ensuring your work meets assignment
expectations.
Misrepresentation of AI-generated work as your own will be reported to the
Aggie Honor System Office. The minimum penalties include:
A grade of 0 on the assignment
A grade of F in the course
Honor Violation Probation
