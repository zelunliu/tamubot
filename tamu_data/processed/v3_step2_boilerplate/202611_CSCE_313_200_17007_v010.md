[16.0pt] Catalog Description
Introduction to Computer Systems. (3-2). Credit 4. Introduction to system support
for application programs, both on single node and over network including OS
application interface, inter-process communication, introduction to system and
network programming, and simple computer security concepts; hands-on lab
assignments. Prerequisite: CSCE 221 with a grade of C or better; grade of C or
better in CSCE 312 or concurrent enrollment in CSCE 350 or ECEN 350.
[16.0pt] Course Prerequisites
[11.0pt bold] Prerequisite/Corequisite(s):  CSCE 221 with a grade of C or better; grade of
C or better in CSCE 312 or concurrent enrollment in CSCE 350/ECEN 350 or
ECEN 350/CSCE 350.
[16.0pt] Course Learning Outcomes
[12.0pt] By the end of the semester, students will be able to utilize basic
[12.0pt] operating-system services and C/C++ system APIs, create multi-
[12.0pt] threaded solutions, implement proper synchronization, and build
[12.0pt] applications with efficient file I/O and memory management.
[12.0pt] Compared to the non-honors version of the class, this course
[12.0pt] provides a more in-depth coverage of synchronization/threading,
[12.0pt] focuses on achieving higher performance, and requires more
[12.0pt] sophisticated code development & debugging.

[12.0pt] and release mode. Solutions with missing files, incorrect project

[12.0pt] not be graded.
[12.0pt] Each homework must be accompanied by a written report (if there
[12.0pt] are multiple parts in the homework, the report applies only to the

[12.0pt] performance analysis requested in the handout. Write in as much
[12.0pt] detail as possible, explain the various observations, and comment
[12.0pt] on the sanity of obtained results.
[12.0pt] This course assumes independent work on each assignment. All
[12.0pt] submissions must be original and developed from scratch. At no
[12.0pt] time can you receive aid from external sources (e.g., current
[12.0pt] students, prior students, material found online, AI tools). The only
[12.0pt] exceptions to this policy are: sample code provided by the
[12.0pt] instructor on the course website, Microsoft help pages, and any
[12.0pt bold] interaction with the TA/instructor.  Any academic dishonesty,
[12.0pt bold] including cheating and plagiarism, will result in an F* for the
[12.0pt bold] course and may lead to expulsion from the university.
