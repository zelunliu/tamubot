[16.0pt] Catalog Description
Introduction to Computer Systems. (3-2). Credit 4. Introduction to system support
for application programs, both on single node and over network including OS
application interface, inter-process communication, introduction to system and
network programming, and simple computer security concepts; hands-on lab
assignments. Prerequisite: CSCE 221 with a grade of C or better; grade of C or
better in CSCE 312 or concurrent enrollment in CSCE 350 or ECEN 350.
[16.0pt] Course Prerequisites
Prerequisite/Corequisite(s):  CSCE 221 with a grade of C or better; grade of C or
better in CSCE 312 or concurrent enrollment in CSCE 350/ECEN 350 or
ECEN 350/CSCE 350.
[16.0pt] Course Learning Outcomes
By the end of the semester, students will be able to utilize basic operating-system
services and C/C++ system APIs, create multi-threaded solutions, implement proper
synchronization, and build applications with efficient file I/O and memory
management.

[16.0pt] AI Statement
With the rise of artificial intelligence (AI) technologies, programming and problem-
solving practices are evolving. While these tools can assist with brainstorming,
[11.0pt bold] debugging, or exploring alternative approaches, they  do not replace the essential
[11.0pt bold] skills of designing, writing, and testing your own code . Nor do they substitute for
the critical thinking required to analyze problems, evaluate solutions, and apply
course concepts independently.
In this course, you are permitted to use generative AI tools (e.g., ChatGPT, GitHub
Copilot, Google Gemini) to support your learning, under the following conditions:
[11.0pt bold] AI may be used as a supplement, not a substitute.  You may consult AI for
hints, debugging suggestions, or explanations, but you are expected to write
and understand your own code.
[11.0pt bold] You must cite AI assistance.  If AI tools inform your work (e.g., generating a
code snippet, suggesting an algorithm, or helping you debug), you must
clearly document this in comments within your code (e.g.,  # AI tool
(ChatGPT) suggested this loop structure   ).
[11.0pt bold] AI should not complete your assignments for you.  Submitting work
primarily written by AI is considered academic dishonesty. You are responsible
for understanding and being able to explain every line of code you submit.
[11.0pt bold] Limit reliance.  No more than 30% of an assignment’s code may be directly
AI-generated. The majority of the work must reflect your own effort and
understanding.
[11.0pt bold] Important reminders:
Generative AI may produce incorrect or inefficient code. Always test and verify
any AI-assisted solution before using it.
AI tools may generate code that does not follow best practices or course
requirements. You are responsible for ensuring your work meets assignment
expectations.
Misrepresentation of AI-generated work as your own will be reported to the
[11.0pt bold] Aggie Honor System Office . The minimum penalties include:
[11.0pt bold] A grade of 0 on the assignment
[11.0pt bold] A grade of F* in the course
[11.0pt bold] Honor Violation Probation
