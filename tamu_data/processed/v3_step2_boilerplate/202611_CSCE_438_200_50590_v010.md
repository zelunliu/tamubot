[16.0pt] Catalog Description
Distributed Systems. (3-0). Credit 3. Principles and techniques for engineering
distributed systems with topics including communication, concurrency,
programming paradigms, naming, managing shared state, caching, synchronization,
reaching agreement, fault tolerance, security, middleware and distributed
applications; design, implement and debug large software systems. Prerequisite:
CSCE 313; junior or senior classification, or approval of instructor.
[16.0pt] Additional Course Details
The objectives for this course are for students to gain an understanding of the
principles and practices (e.g., locking, concurrency, scheduling, communication
across networks, fault tolerance, protocols, remote procedure calls, file sharing,
reliable system design, load balancing, protection and security) behind the design of
distributed processing systems, such as Google, Facebook, Amazon, Yahoo and
Microsoft, and to gain practical experience designing, implementing, and debugging
distributed systems. Honors students will evaluate newly proposed distributed
systems technologies, e.g., blockchain, distributed ledger.

[16.0pt] Course Learning Outcomes
Upon completion of this course, the learner will be able to:
Implement and structure distributed systems programs
Write computer programs that interoperate using well-defined protocols
Employ and create common paradigms (remote procedure calls and
message oriented middleware) for easing the task of distributed systems
programming
Debug highly concurrent code that spans multiple programs running on
multiple machines
Reason about distributed algorithms, locking, synchronization and
concurrency, scheduling and replication
Evaluate newly proposed distributed systems technologies (e.g., blockchain,
distributed ledgers) from the perspective of suitability to a particular
application.
